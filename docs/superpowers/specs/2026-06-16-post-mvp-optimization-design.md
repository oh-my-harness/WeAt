# WeAt Post-MVP 优化设计

## 1. 背景

WeAt Web MVP 已完成 Phase 1~3（聊天 + 浏览器端 AI Agent + 知识库），现进入产品打磨阶段。核心问题：开发循环依赖云端部署验证、移动端体验粗糙、AI 交互流程有摩擦、缺乏用户增长机制。

## 2. 目标

| 优先级 | 目标 |
|--------|------|
| P0 | 本机可完整运行开发环境，不再依赖云端测试 |
| P1 | 移动端体验对标微信，有原生 App 手感 |
| P1 | 前端视觉层次清晰，AI 入口可发现 |
| P1 | AI 草稿交互流畅，流式输出消除等待感 |
| P2 | PWA 可安装到主屏幕，自助邀请码注册 |

## 3. 路线图

采用**路线 1（稳扎稳打）**，四个 Sprint 顺序执行，每个 Sprint 独立可验证。

```
Sprint 1: 本机开发环境诊断修复
Sprint 2: 移动端 UX（WeChat 式）
Sprint 3: 前端视觉刷新 + AI DraftPanel 重设计 + 流式输出
Sprint 4: PWA + 自助注册（邀请码机制）
```

## 4. 设计

### Sprint 1：本机开发环境

**成功标准：** `./dev.sh` 在开发机跑通，能登录、收发消息、AI 起草可用。

**做法：** 执行 `./dev.sh`，逐步排查报错并修复（依赖、端口、配置等）。无设计决策，直接执行。

---

### Sprint 2：移动端 UX

**布局**
- 底部固定导航（"聊天" / "设置" 两个 Tab）
- 聊天页输入框吸底
- 键盘弹起时输入框随之上移（`dvh` 单位 + `visualViewport` API，解决 iOS Safari 经典键盘遮挡问题）

**消息气泡**
- 自己的消息：靠右蓝色气泡
- 对方消息：靠左灰色气泡 + 头像
- 和微信会话气泡风格一致

**原生手感**
- 按钮点击触感反馈（`active:scale-95`）
- 页面切换滑入动画
- 列表滚动顺滑（`-webkit-overflow-scrolling: touch`）

**房间列表**
- 每行：房间名 + 最后一条消息预览 + 时间戳（微信会话列表布局）

**AI 交互（移动端）**
- 触发方式：点消息旁按钮 或 输入斜杠命令
- 触发后跳转独立 AI 全屏页
- 全屏页布局：
  - 上方：上下文消息（可折叠）
  - 中间：AI 草稿内容（流式输出逐字显示）
  - 底部：修改指令输入框 + 重新生成 + 发送按钮
- 点"发送"后自动返回聊天页

---

### Sprint 3：前端视觉刷新 + AI 体验

**视觉风格**
- 整体：简洁白底，主色 `#07C160`（微信绿），去掉深色技术感调性
- 字体层级：房间名 16px 加粗 / 消息预览 14px 灰色 / 时间戳 12px 浅灰
- 桌面端保留侧栏布局，视觉语言与移动端统一

**信息层次**
- 未读消息：绿色角标
- 当前选中房间：高亮
- AI 功能入口：明显的悬浮按钮（FAB），而非隐藏在消息旁的小图标

**DraftPanel 重设计（桌面端）**
- 从右侧滑出的面板，不遮挡消息列表
- 面板分三区：
  1. 上方：参考消息上下文（默认折叠）
  2. 中间：草稿文本（可直接编辑）
  3. 底部：修改指令输入框 + 重新生成 + 一键发送

**AI 流式输出**
- 后端 LLM 调用改为 SSE / streaming
- 前端草稿内容逐字显示
- 消除"等待 → 突然全出来"的割裂感

---

### Sprint 4：PWA + 自助注册

**PWA**
- `manifest.json`：App 名称、图标、主题色 `#07C160`
- Service Worker：基础缓存策略，离线时显示"网络不可用"而非白屏
- iOS / Android 主屏幕安装引导提示
- 效果：用户可安装到主屏幕，打开无浏览器地址栏，图标与原生 App 无异

**自助注册（邀请码机制）**
- 前端：新增注册页（用户名 + 密码 + 邀请码）
- 后端：`POST /api/register`，验证邀请码后通过 Matrix API 创建用户
- 邀请码：管理员在 Admin 面板生成，固定字符串（保持私有团队属性，防止任意注册）
- Admin 面板：新增"生成邀请码"入口

## 5. 额外发现的优化方向（未纳入本期）

以下方向在讨论中提及，可作为后续 backlog：

- **消息搜索**：在房间历史中全文检索
- **推送通知**：PWA push notification，用户不在 App 时收到新消息提醒
- **多语言支持**：如面向更广泛用户群
